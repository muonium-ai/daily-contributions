## Goal
Track the last commit date on each repo, classify repos by activity status, and display status alongside each repo in the timeline and as a cumulative summary.

## Acceptance Criteria
- [ ] `repos` table has a `last_commit_date` column
- [ ] `index_loc.py` populates `last_commit_date` using `git log -n 1 --pretty=format:%aI`
- [ ] Each repo is classified by days since last commit:
  - **active**: ≤ 7 days
  - **stale**: 8–30 days
  - **inactive**: 31–90 days
  - **dormant**: > 90 days
- [ ] Module Growth Timeline shows status alongside each repo
- [ ] New "=== Repo Activity Summary ===" section shows counts per status
- [ ] All scripts work via `make report`

## Notes
Classification is based on calendar days between today and the repo's last commit date (author date). Repos with no commits default to dormant.

Example output:
```
=== Module Growth Timeline ===
2025-01-15  MuonJS        https://github.com/...  [active]
2025-02-03  MuonVec       https://github.com/...  [stale]
2025-03-10  OldProject    https://github.com/...  [dormant]

=== Repo Activity Summary ===
active (≤7 days):    12
stale (8-30 days):    5
inactive (31-90 days): 3
dormant (>90 days):   8
total: 28
```

## Implementation Plan
- [ ] Add `last_commit_date` column to `repos` table schema and migration in `init_db`
- [ ] In `index_loc.py` main(), fetch last commit date via `git log -n 1 --pretty=format:%aI` and store it
- [ ] In `report_generator.py`, add a `classify_activity(last_commit_date)` helper
- [ ] Update `print_module_timeline()` to show status tag per repo
- [ ] Add `print_activity_summary()` with counts per status category
- [ ] Run `make clean && make` to verify
