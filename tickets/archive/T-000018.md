## Goal
Stop `tee` from masking a crash in `report_generator.py` so `make report` exits non-zero and doesn't leave a half-written report file.

## Acceptance Criteria
- [ ] `make report` exits non-zero if `report_generator.py` raises
- [ ] On failure, `reports/YYYY-MM-DD.txt` is either absent or untouched (no partial overwrite)
- [ ] Successful runs still produce the same report content

## Notes
[Makefile:21](Makefile#L21) pipes through `tee`, which always exits 0. Either set `SHELL := bash` with `.SHELLFLAGS := -eu -o pipefail -c`, or write to a temp file and `mv` only on success.
