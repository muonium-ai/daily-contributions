## Goal
Store each repo's name, URL, and first commit date in a `repos` table so we can track when each module/component first appeared.

## Acceptance Criteria
- [ ] New `repos` table with columns: `name`, `path`, `url`, `created_date`, `updated_at`
- [ ] `index_loc.py` (or a new indexer) populates the table from `config/repos.txt` using `git log --reverse -n 1 --pretty=format:%aI`
- [ ] Table is refreshed on each run (INSERT OR REPLACE)
- [ ] `make index` triggers the repo metadata indexing

## Notes
`report_generator.py` already has `get_repo_created_date()` — reuse that logic. Move repo metadata collection into the indexing phase so the report just reads from DB.

## Implementation Plan
- [ ] Add `repos` table schema to `init_db` in `index_loc.py`
- [ ] After daily LOC indexing, iterate repos and store name, path, url, created_date
- [ ] Verify with `make clean && make`
