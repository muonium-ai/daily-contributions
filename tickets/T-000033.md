## Goal
Add a tool that discovers git worktrees across the user's tracked repos and offers safe options to clean them up — stale (path missing), prunable, on deleted/merged branches, or selected by glob — defaulting to dry-run so it is safe to run on a CI server.

## Acceptance Criteria
- [ ] `src/daily_contributions/worktrees.py` exposes `list_worktrees(repo_path)`, `classify_worktrees(...)`, `cleanup_worktrees(...)`, and a `main(argv=None)` entry point
- [ ] `tools/clean_worktrees.py` is a thin wrapper that delegates to `daily_contributions.worktrees.main`
- [ ] Discovery uses `git worktree list --porcelain` (parsed in Python — no `git worktree list` text scraping); falls back gracefully when a repo has only the main worktree
- [ ] By default, accepts a path of repos to scan (or reads `config/repos.txt` if `--from-config` is given) and prints a per-repo summary: total worktrees, stale (path missing), prunable, on-merged-branch, with-uncommitted-changes
- [ ] Cleanup is **dry-run by default**; `--apply` actually performs removals; `--force` is required to remove worktrees with uncommitted changes
- [ ] Cleanup categories are selectable via flags (e.g. `--prune-stale`, `--remove-merged`, `--match <glob>`); never removes the main worktree
- [ ] Tests cover the porcelain parser and the classifier with fixture data (no real `git` calls)
- [ ] `uv run pytest tests/` is green

## Notes
**Safety rails (important — this tool destroys filesystem state):**
- Default = dry-run. Print exactly what *would* be removed, grouped by repo and category.
- Refuse to remove a worktree that has uncommitted/untracked changes unless `--force` is passed.
- Never touch the primary worktree (`bare = false` and `branch = HEAD` of the repo).
- Always run `git worktree prune` *after* removals so `.git/worktrees/<name>` metadata stays consistent.

**Discovery strategy:** prefer parsing `git worktree list --porcelain`. The format is stable and emits one `worktree <path>` block per worktree, with `HEAD`, `branch`, optional `bare`, `detached`, and `prunable` keys.

**"Merged" classification:** a worktree's branch is "merged" if `git branch --merged <default-branch>` includes it. Default branch detection: `git symbolic-ref refs/remotes/origin/HEAD` → fallback to `main` then `master`. Skip the merged check (and report `unknown`) if none resolves.

**Why this fits:** complements `find_node_modules` and `image_sequence_detection` as a CI-server cleanup utility, and reuses the `config/repos.txt` produced by `tools/find_repos.py`.
