## Goal
Detect which repos use MuonTickets and report ticket counts (archived/total) per repo and as a summary.

## Acceptance Criteria
- [ ] For each repo, check if a `tickets/` folder exists
- [ ] If yes, count `*.md` files in `tickets/archive/` (implemented tickets) and `tickets/*.md` (open tickets)
- [ ] Show ticket usage alongside each repo in the per-repo summary (e.g., `tickets: 5 done / 8 total` or `tickets: none`)
- [ ] New "=== MuonTickets Summary ===" section at end of report listing:
  - Each repo that uses MuonTickets with its archived and total ticket counts
  - Grand totals across all repos

## Notes
A repo "uses MuonTickets" if it has a `tickets/` directory. Archived tickets live in `tickets/archive/*.md`. Open/active tickets are `tickets/*.md` (excluding non-ticket files like `ticket.template` and `last_ticket_id`).

Example output:
```
=== MuonTickets Summary ===
daily-contributions:  11 done / 11 total
muon-js:              3 done / 5 total
HiggsAgent:           none

repos using MuonTickets: 2
total tickets implemented: 14
total tickets: 16
```

## Implementation Plan
- [ ] Add helper function `get_ticket_counts(repo_path)` that checks for `tickets/` dir, counts archive/*.md and tickets/T-*.md files
- [ ] In `print_repo_summary()`, show ticket info per repo
- [ ] Add `print_tickets_summary()` that queries all repos and prints the MuonTickets summary section
- [ ] Call from `__main__` block
- [ ] Run `make clean && make` to verify
