## Goal
Make `index_loc.py` re-index today's row on every run so commits made later in the day are picked up without `make clean`.

## Acceptance Criteria
- [ ] After indexing, running `make index` again the same day re-scans today's date and updates totals
- [ ] Backfill behavior unchanged: a fresh DB still indexes from `config/start-date.txt`
- [ ] Subsequent days are unaffected (no double-counting)

## Notes
`get_start_date` in [index_loc.py:21-29](index_loc.py#L21-L29) returns `MAX(date) + 1 day`, which skips today on a second same-day run. Returning `MAX(date)` (without the `+1`) is safe because the insert uses `INSERT OR REPLACE` keyed on `date`.
