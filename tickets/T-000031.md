## Goal
Once the cleanup tools live in the package, expose them as first-class CLIs alongside `repos`/`index`/`report`: register them in `[project.scripts]` in `pyproject.toml` and add Makefile targets so they fit the existing developer workflow.

## Acceptance Criteria
- [ ] `pyproject.toml` adds `[project.scripts]` entries (e.g. `find-node-modules = "daily_contributions.node_modules_scanner:main"`, `find-image-sequences = "daily_contributions.image_sequence:main"`)
- [ ] `Makefile` adds `node-modules` and `image-sequences` targets that accept a `ROOT_DIR=` argument and shell out to the wrappers in `tools/`
- [ ] `make node-modules ROOT_DIR=...` and `make image-sequences ROOT_DIR=...` both run end-to-end and print the expected report
- [ ] `README.md` documents the two new make targets in the same style as the existing ones
- [ ] `uv run pytest tests/` is green

## Notes
Keep the Makefile targets consistent with the existing `repos`/`index`/`report` conventions (uppercase variable name, `uv run tools/...`). Script names in `[project.scripts]` are entry-points exposed when the package is installed; the make targets keep the `uv run tools/*.py` invocation style for parity with the rest of the file.
