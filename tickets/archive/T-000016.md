## Goal
Compute `avg commits` from per-day commit totals so it's comparable to `avg additions`/`avg deletions`.

## Acceptance Criteria
- [ ] `avg commits` in the "Non-zero days averages" section is the average of `daily_loc.commits` over non-zero days
- [ ] Value is no longer derived from `git rev-list --count HEAD` aggregates
- [ ] Reported value matches a manual sum/avg of the daily report rows

## Notes
[report_generator.py:207](report_generator.py#L207) currently divides `total_commits` (lifetime, from `git rev-list HEAD` per repo in `print_repo_summary`) by `days` (non-zero days in the indexed window). Pull commits from `daily_loc` alongside the other averages in `get_non_zero_day_averages`.
