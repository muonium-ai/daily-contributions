## Goal
Have `print_repo_summary` source URL, created date, and last-commit date from the `repos` table that `index_loc.py` already populates.

## Acceptance Criteria
- [ ] Report no longer shells out to `git config remote.origin.url` or `git log` for per-repo metadata
- [ ] Output for a repo with stale on-disk state still reflects what was last indexed
- [ ] `make report` runtime drops noticeably on a many-repo config

## Notes
`index_loc.py` already writes `url`, `created_date`, `last_commit_date` into `repos` ([index_loc.py:179-182](index_loc.py#L179-L182)); the report re-fetches them in [report_generator.py:158-166](report_generator.py#L158-L166).
