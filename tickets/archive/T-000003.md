## Goal
Fix the `read_lines()` helper in `index_loc.py` and `report_generator.py` to properly close file handles using a `with` statement.

## Acceptance Criteria
- [ ] `read_lines()` in `index_loc.py` uses a `with open(...)` context manager
- [ ] `read_lines()` in `report_generator.py` uses a `with open(...)` context manager
- [ ] No functional change — same return values

## Notes
Current code `return [l.strip() for l in open(path) if l.strip()]` opens a file without closing it, leaking the handle. Should be:
```python
def read_lines(path):
    with open(path) as f:
        return [l.strip() for l in f if l.strip()]
```

## Implementation Plan
- [ ] Update `read_lines()` in `index_loc.py` to use `with open(...) as f:`
- [ ] Update `read_lines()` in `report_generator.py` to use `with open(...) as f:`
- [ ] Run `uv run index_loc.py` and `uv run report_generator.py` to verify no regressions
