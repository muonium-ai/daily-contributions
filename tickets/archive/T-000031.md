## Goal
Once the cleanup tools live in the package, expose them as first-class CLIs alongside `repos`/`index`/`report`: register them in `[project.scripts]` in `pyproject.toml` and add Makefile targets so they fit the existing developer workflow.

## Acceptance Criteria
- [x] `pyproject.toml` adds `[project.scripts]` entries for `find-node-modules`, `find-image-sequences`, and `clean-worktrees`
- [x] `Makefile` adds `node-modules`, `image-sequences`, and `worktrees` targets that accept a `ROOT_DIR=` argument and shell out to the wrappers in `tools/`
- [x] `make node-modules ROOT_DIR=...`, `make image-sequences ROOT_DIR=...`, and `make worktrees ROOT_DIR=...` all run end-to-end and print the expected report
- [x] `README.md` documents the three new make targets in the same style as the existing ones
- [x] `uv run pytest tests/` is green

## Notes
Keep the Makefile targets consistent with the existing `repos`/`index`/`report` conventions (uppercase variable name, `uv run tools/...`). Script names in `[project.scripts]` are entry-points exposed when the package is installed; the make targets keep the `uv run tools/*.py` invocation style for parity with the rest of the file.

Scope expanded post-T-000033 to also wire `clean-worktrees`. The worktree make target intentionally omits `--apply` so the default-dry-run behaviour is preserved; users pass `--apply` directly to the underlying tool when they want to act.
