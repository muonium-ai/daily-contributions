## Goal
Extract duplicated config path constants (`DB_PATH`, `REPOS_FILE`, `EMAILS_FILE`) into a shared module to eliminate drift risk.

## Acceptance Criteria
- [ ] A shared module (e.g., `config.py` or `constants.py`) defines `DB_PATH`, `REPOS_FILE`, `EMAILS_FILE`
- [ ] `index_loc.py` and `report_generator.py` import from the shared module instead of defining their own
- [ ] All scripts continue to work via `make all`

## Notes
Currently `DB_PATH = "data/contributions.db"`, `REPOS_FILE = "config/repos.txt"`, `EMAILS_FILE = "config/emails.txt"` are duplicated in both `index_loc.py` and `report_generator.py`. If one is updated and the other isn't, they'll silently diverge.

## Implementation Plan
- [ ] Create a `constants.py` (or `config.py`) file with the shared paths
- [ ] Update `index_loc.py` to import from the shared module
- [ ] Update `report_generator.py` to import from the shared module
- [ ] Run `make all` to verify end-to-end
