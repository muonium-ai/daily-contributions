## Goal
Lay the foundation for a `src/` + `tools/` layout: create an empty `src/daily_contributions/` package and wire `pyproject.toml` so `uv sync` installs it editable into the venv.

## Acceptance Criteria
- [ ] `src/daily_contributions/__init__.py` exists (empty or single `__all__`)
- [ ] `tools/` directory exists (empty placeholder file or `.gitkeep` is fine)
- [ ] `pyproject.toml` has a `[build-system]` block (`hatchling`) and tells hatch to package `src/daily_contributions`
- [ ] `uv sync` succeeds and `uv run python3 -c "import daily_contributions"` works
- [ ] No existing module is moved yet — that comes in follow-up tickets
- [ ] Existing `make report` and `uv run pytest tests/` still pass

## Notes
Use the standard PEP 621 + hatchling src layout:
```toml
[build-system]
requires = ["hatchling>=1.24.0"]
build-backend = "hatchling.build"

[tool.hatch.build.targets.wheel]
packages = ["src/daily_contributions"]
```
Subsequent tickets (T-000024..T-000027) move modules into the package one by one so each move stays small and reviewable.
